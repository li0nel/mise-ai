Implementation Specification

Screen Overview

The Progressive Accordion pattern presents all recipe-builder question categories as a vertical accordion list from the moment analysis completes. One question is expanded at a time (the current active question), while completed questions collapse to a single row showing the answer. Pending questions appear as dimmed collapsed rows. Users can tap any completed row to revise their answer.

Layout Hierarchy

  1. StatusBar (44px) — system time, signal icons
  2. AppHeader (48px) — back arrow + "mise." logo
  3. ScrollView (flex: 1, padding: 16px)
    1. AnalysisChip — centered, tappable, shows recipe name + source count
    2. ProgressBar — 6 dots showing completion state (appears after first answer)
    3. Accordion — vertical stack of AccordionRow components
      1. AccordionRow × 6 (one per question category)
        1. RowHeader: number badge, label, answer text (if completed), chevron
        2. PillArea (conditional): question text + pill options grid
    4. CelebrationCard (conditional, after all answered) — title, subtitle, CTAs

Animation & Transitions

  • Expand/Collapse: LayoutAnimation.configureNext(LayoutAnimation.Presets.easeInEaseOut) — 250ms ease-in-out for height changes
  • Chevron rotation: Animated.timing 0° ↔ 180°, 200ms
  • Pill selection: Instant border/background swap via className conditional
  • Row state change: Background/border color transitions via 200ms CSS transition equivalent
  • Celebration card: Slide-up + fade-in, 300ms spring
  • Auto-scroll: After answering, scrollToIndex brings the newly-active row into view

Interaction Rules

  • Forward flow: Selecting a pill on the active row immediately collapses it (completed state) and expands the next pending row.
  • Revision: Tapping any completed row header opens it in "revising" state (amber). The active row remains open. Selecting a different pill in the revising row re-collapses it with the new answer.
  • Multi-select (Dietary): Tapping pills toggles them. A "Done" or "Next" button appears in the pill area to confirm and advance.
  • Single-select (all others): Tapping a pill immediately selects it and auto-advances.
  • Pre-checked pills: Shown with dashed brand border. The AI detected a likely preference (e.g., nut allergy). User can deselect.
  • URL flow: "Customize This Recipe" button reveals the accordion below the preview card. Categories may be pre-filled from the parsed recipe.

Accessibility

  • Each AccordionRow header has accessibilityRole="button" and accessibilityState={{ expanded }}
  • Pills use accessibilityRole="radio" (single-select) or "checkbox" (multi-select) with accessibilityState={{ selected }}
  • Screen reader announces: "Dietary Needs, completed, Nut-free. Tap to revise." for completed rows
  • Progress bar: accessibilityLabel="Step 4 of 6"
  • Focus management: when a row expands, focus moves to the first pill

Responsive Notes

  • Designed for 390px width (iPhone 14/15). Scales naturally to 375px (iPhone SE/13 mini) and 430px (Pro Max).
  • Pill grid uses flexWrap: wrap so pills reflow on narrower screens.
  • Accordion rows have a minimum touch target of 44px height.
  • Scroll view ensures all content is accessible when many rows are expanded (revision scenario).
  • Safe area insets handled by Expo's SafeAreaView.

What the Mock Does NOT Show

  • Loading state: Skeleton rows while analysis is in progress (before Frame 1).
  • Error handling: What happens if analysis fails or times out.
  • Multi-select confirmation: The "Done" button in the Dietary Needs pill area for multi-select scenarios.
  • Scroll behavior: Auto-scroll to newly active row after answering.
  • Haptics: Light impact feedback on pill selection, medium on row completion.
  • Analytics events: builder_question_answered, builder_answer_revised, builder_completed.
  • Deep linking: Sharing a builder state via URL for collaborative recipe planning.
  • Keyboard/Voice: No free-text input shown. Voice input for accessibility is not depicted.
  • Recipe generation: The loading/streaming state between "all answered" and the celebration card appearing.
