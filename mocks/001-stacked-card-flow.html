Implementation Specification

1. Screen Overview

  • Frame 1 (Analysis): Pulsing shimmer card signals active source analysis; no user input required.
  • Frame 2 (Dietary): Analysis collapses to chip; first multi-select question card appears with pre-filled saved preference.
  • Frame 3 (Mid-Flow Q4): Three answered chips stack above the current single-select spiciness question.
  • Frame 4 (Recipe Ready): Full progress bar, confetti, and celebration card with primary "View Recipe" CTA.
  • Frame 5 (URL Preview): Pasted URL resolves to rich preview card with dual action paths.
  • Frame 6 (Customize Fork): URL-sourced recipe enters the same stacked card customization flow.

5. Responsive Notes

  • This mock targets 390px mobile viewport (iPhone 14/15 logical width).
  • On tablet (768px+): phone shells could display in a 2-up grid; content area gains horizontal padding.
  • On desktop (1024px+): phone shells display in a horizontal scroll or flex row as shown here.
  • The actual app viewport is always 390px equivalent due to React Native / Expo rendering.
  • Touch targets: all option pills meet 44px minimum touch target (10px vertical padding + 14px font + line-height achieves ~44px).

6. What the Mock Does NOT Show

  • Loading skeletons: Skeleton placeholders for chip collapse animation and card transitions.
  • Offline state: Error card with retry button when network is unavailable.
  • Keyboard navigation: Tab order through pills, Enter/Space to select, Escape to go back.
  • Screen reader announcements: Live region updates for progress bar, question changes, and selection confirmations.
  • Animation durations: Card slide-up 300ms ease-out, pill selection 200ms cubic-bezier, progress bar 400ms cubic-bezier, celebration pop 400ms spring.
  • Error states: Failed analysis, network timeout, server error recovery flows.
  • Haptic feedback: Light impact on pill selection, medium impact on "Continue", success notification on recipe ready.
  • Chip re-open interaction: Tapping a collapsed answer chip expands it back into a full question card (reverse of collapse animation).
  • Skip flow: "Skip" affordance on optional questions (e.g., dietary needs).
