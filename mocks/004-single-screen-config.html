Implementation Spec — Variation 004: Single-Screen Configurator

Screen Overview

A single, non-scrolling (ideally) configurator screen where all recipe preference sections are visible at once. The user taps pills to make selections within each section. A primary CTA at the bottom reflects completion progress and becomes fully active when all sections are filled.

Layout Hierarchy

  • StatusBar (44px) — time, network, battery indicators
  • AppHeader (48px) — back chevron + "mise." wordmark
  • AnalysisChip — emoji + recipe name + source count or domain
  • SectionsContainer (flex:1, gap:14px, pad:0 20px)
    • Section x6 — label + pill group
  • CTAContainer (sticky bottom, pad:12px 20px 24px) — primary action button

Interaction Details

  • Pill tap (single-select section): Deselects any currently selected pill in that section, selects the tapped pill. Sends chat message: "Set [section] to [value]"
  • Pill tap (multi-select section): Toggles the tapped pill. Sends chat message: "Add/Remove [value] from [section]"
  • CTA tap (active): Sends "Build my recipe with these preferences" as a chat message. Triggers the recipe generation flow.
  • Adjust Preferences: Returns to the configurator with all previous selections intact.
  • Save As-Is (URL flow): Accepts all pre-filled values without modification.

Responsive Notes

  • Designed for 390px width (iPhone 14/15). On narrower devices (375px, SE), pills may wrap to additional rows — sections container should scroll if content exceeds viewport.
  • On wider devices or tablets, sections could be arranged in a 2-column grid to reduce vertical height further.
  • CTA container uses safe-area-inset-bottom padding on devices with home indicator.

What the Mock Does NOT Show

  • Loading / building state: Spinner or progress animation on CTA during recipe generation.
  • Error states: Network failure, generation timeout, or API error handling.
  • Section reordering: Sections could be reordered based on recipe context (e.g., "Fish Sauce" for Thai recipes).
  • Keyboard/accessibility: Focus ring styles, screen reader labels, keyboard navigation between pills.
  • Haptic feedback: Pill selection should trigger light haptic on iOS.
  • Animated transitions: Configurator-to-celebration crossfade, pill selection micro-animations.
  • Scroll behavior: On shorter devices, the sections container scrolls with momentum; CTA remains sticky.
  • Saved preferences sync: How user dietary prefs are stored and pre-applied across sessions.
