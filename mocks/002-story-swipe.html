Implementation Specification — Story Swipe (Variation 002)

1. Screen Overview

Frame Screen Purpose Background Tint
1 Analysis Phase Immersive loading state while sources are analyzed Warm gradient (#FDF5EE to #F5DCC6)
2 Dietary Needs Multi-select dietary restrictions Peach (#FDF5EE to #FAE3D3)
3 Spice Level Single-select spice preference Amber (#FDF8EE to #F9E8C6)
4 Revised Answer Change-of-mind flow with toast confirmation Amber (same as Frame 3)
5 Recipe Ready Celebration / completion moment Warm celebration gradient
6 URL Fork Decision Save vs. customize a pasted recipe URL Neutral warm (#FDFAF7 to #EDE2D6)

2. Layout Hierarchy

  • StatusBar — 44px, time + system icons, always visible
  • HeaderBar — Semi-transparent, back arrow + "mise." logo centered + spacer
  • ProgressSegments — Horizontal bar split into N segments (one per question), 3px tall, 4px gap
  • StoryContent — Flex-grow area, vertically and horizontally centered content
  • Each screen fills the entire content area below the progress bar — no scrolling, no secondary content zones

4. State Matrix

Interaction Trigger Visual Change Data Effect
Select pill (multi) Tap pill Border → brand fill, text → white, checkmark appears Toggle item in array (e.g. dietary: ["nut-free"])
Select pill (single) Tap pill Previous deselects, new pill fills brand Replace value (e.g. spice: "hot")
Advance Tap "Next" or swipe left Cross-fade to next question, fill next progress segment, background tint shifts Step index increments
Go back Swipe right or tap back arrow Reverse transition, previous tint Step index decrements
Jump to step Tap a filled progress segment Immediate transition to that step Step index set to tapped segment
Revise answer Change selection on revisited step Toast appears: "Updated! Tap any dot to revisit." Value replaced in state
Complete All questions answered + final "Next" Celebration screen with confetti, all segments filled Build recipe with collected answers

5. Responsive Notes

  • Phone shell is fixed at 390×760px — content is pixel-accurate to device
  • In production, the story view fills 100dvh minus safe-area insets
  • Background tints use CSS custom properties for easy theming per step
  • Pill grid wraps naturally; on narrower devices (320px), pills may stack to fewer per row
  • Large pills (spice) are always full-width, stacked vertically
  • Toast is absolutely positioned and auto-dismisses after 3s (not shown in static mock)
  • Swipe gestures handled via react-native-gesture-handler PanGestureHandler or Animated.event

6. What the Mock Does NOT Show

  • Swipe transitions — cross-fade / slide animations between steps are described but not interactive in this static mock
  • Gesture handling — swipe left/right to advance/retreat
  • Auto-advance — on single-select questions, selecting a pill could auto-advance after a 500ms delay
  • Toast auto-dismiss — toast should disappear after ~3 seconds
  • Skip / "None" option — some questions may need a skip affordance
  • Loading states — between analysis completion and first question
  • Haptic feedback — light impact on pill selection, medium on step advance, success on celebration
  • Accessibility — role="radiogroup" / role="checkbox", aria-current for progress, reduced-motion preferences for confetti
  • Deep link handling — Frame 6 URL fork is triggered externally; the entry point (share sheet, clipboard detection) is not shown
  • Questions 2, 3, 5 — only dietary (Q1) and spice (Q4) are mocked; remaining questions (servings, cook time, equipment, garnish) follow identical layout patterns with their own tints
