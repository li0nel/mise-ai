Implementation Spec — Bottom Sheet Cascade

Screen Overview

The Bottom Sheet Cascade pattern uses sequential bottom sheets to guide users through recipe customization. Each question appears as a modal sheet sliding up from the bottom, while previous answers accumulate as compact chips in a horizontally scrollable row above the active sheet. The main content area behind the sheet shows a dimmed/blurred previous context.

Layout Hierarchy

  • Layer 0 — Status Bar: Fixed 44px, always visible, z-index 20
  • Layer 1 — App Header: Fixed 52px, back arrow + "mise." logo, z-index 20
  • Layer 2 — Main Content Area: Fills remaining space below header. Shows analysis state or celebration content.
  • Layer 3 — Backdrop Overlay: rgba(0,0,0,0.3), z-index 5. Appears when a bottom sheet is active. Tapping dismisses the sheet (returns to previous question or cancels).
  • Layer 4 — Answer Chips Row: Horizontal flex, z-index 10, positioned between header and sheet. Overflow-x scroll, gap 8px. Each chip is tappable to revise that answer.
  • Layer 5 — Bottom Sheet: z-index 15, white elevated background, top border-radius 20px. Contains drag handle, question, pills, and primary CTA.

Animation Spec

  • Sheet enter: translateY(100%) → translateY(0), 350ms cubic-bezier(0.32, 0.72, 0, 1) (iOS spring curve)
  • Sheet exit: translateY(0) → translateY(100%), 250ms ease-in
  • Backdrop fade: opacity 0 → 1, 200ms ease-out (enter), 150ms ease-in (exit)
  • Chip appear: scale(0.9) + opacity 0 → scale(1) + opacity 1, 200ms ease-out
  • Pill select: scale(0.95) → scale(1), 150ms with haptic feedback
  • Source items: staggered translateY(8px) + opacity 0, 400ms each, 600ms delay between items
  • Progress bar: width 10% → 65%, 2s ease-out

Gesture Spec

  • Sheet drag down: Pan gesture on sheet; if velocity > 500 or drag > 40% sheet height, dismiss sheet (go back one question)
  • Sheet drag up: No effect (already at max height)
  • Backdrop tap: Dismiss current sheet, return to previous state
  • Chip tap: Open revision sheet for that answer; chip highlights in terracotta
  • Pill tap: Toggle selection state with haptic feedback
  • Chips row scroll: Horizontal swipe, momentum-based, no scroll indicators

Responsive Notes

  • Phone shell is fixed at 390x760px (iPhone 14/15 viewport)
  • Bottom sheet heights are percentage-based (52-68%) and adapt to content
  • On taller devices (>812pt), sheet maintains percentage heights; on shorter devices, sheet takes up more space (max 85%)
  • Chips row horizontally scrolls; no wrapping to prevent vertical growth
  • Safe area inset at bottom: 34px padding accounts for home indicator
  • On landscape or tablet, bottom sheet converts to centered card (max-width 420px) with backdrop still full-screen

What the Mock Does NOT Show

  • Drag-to-dismiss gesture: Interactive sheet dragging, rubber-band effect at top, velocity-based snap-to-dismiss threshold
  • Sheet transition animation: Spring-based slide-up/down between questions, cross-fade of content within the sheet
  • Multi-select vs single-select logic: Dietary allows multi-select; protein and spice are single-select. The mock shows both but does not enforce the constraint
  • Haptic feedback: Light impact on pill selection, medium impact on sheet snap, success notification on recipe ready
  • Keyboard avoidance: Not applicable (no text input in this flow)
  • Loading state between questions: Brief (200ms) skeleton shimmer in sheet while next question loads from LLM
  • Error/retry state: Network failure banner at top, retry button in sheet
  • Skip functionality: Each question can be skipped; "Skip" text button below primary CTA, not shown in mock
  • Accessibility: Sheet announced as modal dialog, focus trap within sheet, VoiceOver rotor for chips, reduce-motion preference disables spring animations
  • Deep link handling: URL flow shows the entry point but not the URL parsing, validation, or fallback states for unsupported domains
